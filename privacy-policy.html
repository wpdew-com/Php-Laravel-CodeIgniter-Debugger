Privacy Policy

Last Updated: February 11, 2026

TL;DR: This extension does NOT collect, store, or transmit any personal data. Everything happens locally in your browser. No tracking. No analytics. No external servers.

1. Introduction

Laravel & CodeIgniter Debugger ("the Extension") is a developer tool designed for local web development. We are committed to protecting your privacy and being transparent about our practices.

2. Data Collection

The Extension does NOT collect, store, or transmit any personal data.

We do not have any servers, databases, or analytics systems. There is no tracking code in the extension.

3. What Data is Processed Locally

The Extension processes the following technical debugging data LOCALLY in your browser:

  • HTTP response headers from your local development server
  • URL of requests (localhost, 127.0.0.1, *.test, *.local domains only)
  • Number of SQL queries executed
  • Page execution time (in milliseconds)
  • Memory usage statistics (in megabytes)
  • Framework version information (Laravel/CodeIgniter version)
  • Route names and controller information
  • HTTP request methods (GET, POST, etc.)

This data:

  • ✓ Stays on your device
  • ✓ Is never sent to external servers
  • ✓ Is automatically deleted when you close browser tabs
  • ✓ Is only visible to you
  • ✓ Can be cleared manually using the "Clear" button

4. Chrome Permissions and Their Usage

4.1. Storage Permission

Used to temporarily cache debugging information for active browser tabs in chrome.storage.local. This allows you to maintain request history (last 100 requests) when reopening the Side Panel.

✗ No personal data is stored
✗ No browsing history is saved
✓ Only technical debugging metrics
✓ Cleared automatically when tabs close

4.2. WebRequest Permission

Used to intercept and read HTTP response headers from web requests using chrome.webRequest.onCompleted.

The extension ONLY reads response headers. It does not:

  • ✗ Modify requests
  • ✗ Modify responses
  • ✗ Block requests
  • ✗ Intercept request bodies
  • ✗ Access cookies or authentication tokens

4.3. SidePanel Permission

Used to display debugging information in Chrome's Side Panel interface. This provides a clean, non-intrusive debugging experience without covering website content.

4.4. Host Permissions

The extension requires access to local development domains only:

  • http://localhost/* - Local development server
  • http://127.0.0.1/* - Local development server (IP address)
  • http://*.test/* - Laravel Valet/Homestead domains
  • http://*.local/* - Common local development domains
  • https://*/* - Local HTTPS development environments

These permissions are ONLY used to read response headers for debugging purposes. The extension is designed for local development and should not be used on production websites.

5. No Third-Party Access

No data is shared with third parties. The extension does not use:

  • ✗ Google Analytics or any analytics service
  • ✗ Tracking pixels or beacons
  • ✗ External APIs or services
  • ✗ Advertisement networks
  • ✗ Social media integration

6. No Remote Code Execution

The extension does not load or execute any remote code. All JavaScript, HTML, and CSS files are included directly in the extension package. No CDNs, no external scripts, no dynamic code loading.

7. Local Development Only

This extension is designed exclusively for local web development. It works with:

  • localhost development servers
  • Laravel Valet domains (*.test)
  • Local network domains (*.local)
  • Docker containers on localhost

Important: Never enable debug headers on production websites. This extension should only be used in development environments.

8. User Control

You have full control over the extension:

  • ✓ Disable: Turn off the extension at any time in chrome://extensions/
  • ✓ Clear Data: Use the "Clear" button in the Side Panel to remove stored debugging data
  • ✓ Uninstall: Completely remove the extension and all its data
  • ✓ Inspect Code: View the extension's source code in chrome://extensions/ (Developer mode)

9. Updates and Changes

We may update this Privacy Policy from time to time. Any changes will be reflected in the "Last Updated" date at the top of this page. Continued use of the extension after changes constitutes acceptance of the updated policy.

10. Children's Privacy

This extension is a professional developer tool and is not intended for use by children under 13. We do not knowingly collect any information from children.

11. Open Source

The extension's source code is available for review. You can verify that it does not collect or transmit any data by inspecting the code yourself.
